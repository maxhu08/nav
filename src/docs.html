rules

rules.urls.mode

Chooses which URL rules list is active.

`blacklist` blocks matching actions from `rules.urls.blacklist`.
`whitelist` only allows matching actions from `rules.urls.whitelist`.

rules.urls.blacklist / rules.urls.whitelist

These fields define URL-based rules. Both fields use the same syntax, and `rules.urls.mode` decides which one is active.

* <regex-pattern>

Starts a new rule block. The `* ` prefix is required before every website regex pattern.


+ <action1> <action2> ...

Blocks all actions except the listed action names for URLs matching the regex pattern above.

- <action1> <action2> ...

Allows all actions except the listed action names for URLs matching the regex pattern above.

You can list any number of action names on a `+` or `-` line.

A `+` or `-` line must go on the next consecutive line immediately after its `* <regex-pattern>` line.

If a rule block contains only `* <regex-pattern>`, all actions are blocked for matching URLs.

Separate action names with spaces.

Lines starting with # are ignored.


Example:

* ^https?://(www\.)?github\.com/.*$

+ hint-mode-current-tab hint-mode-right-click yank-current-tab-url

In `rules.urls.blacklist`, this blocks every action on GitHub pages except `hint-mode-current-tab`, `hint-mode-right-click`, and `yank-current-tab-url`. In `rules.urls.whitelist`, the same rule would allow only those actions on GitHub pages.

rules.forceNormalMode

Keeps nav in normal mode on page load by rejecting automatic focus into editable fields until you manually focus one.

hotkeys

hotkeys.mappings

Each line maps one key sequence to one action name.

<key-sequence> <action-name>

Put one mapping on each line.

Modifier keys are specified as `<c-x>`, `<m-x>`, and `<a-x>` for ctrl+x, meta+x, and alt+x. Combined modifiers are written like `<c-a-x>` or `<c-m-x>`. Use `<space>` for the space key.

Reserved config characters must use named tokens:
- `<` is `<chevronleft>`
- `>` is `<chevronright>`
- `#` is `<hashtag>`

Actions run in mode context:
- normal mode actions work in regular browsing mode.
- find mode actions (`cycle-match-next`, `cycle-match-prev`) only work while find mode is active.
- watch mode actions (`toggle-fullscreen`, `toggle-play-pause`) only work while watch mode is active.

Default hint bindings are `f`, `F`, and `<a-f>` for current-tab, new-tab, and right-click hint mode.

Right-click hint mode behaves like current-tab hint mode, except it right-clicks the selected target, ignores reserved directives, and always shows the cursor marker.

A key sequence can be reused across different modes (for example `f` for hints in normal mode and fullscreen in watch mode), but duplicate mappings in the same mode are rejected.

Lines starting with # are ignored.

Action syntax highlighting:
- normal mode actions are yellow.
- find mode actions are orange.
- watch mode actions are green.
- invalid tokens are red.

In normal mode:

scroll-left

Scrolls left in the current scrollable area.

scroll-down

Scrolls downward from the current scrollable area.

scroll-up

Scrolls upward from the current scrollable area.

scroll-right

Scrolls right in the current scrollable area.

scroll-to-top

Jumps to the top of the current scrollable area.

scroll-to-bottom

Jumps to the bottom of the current scrollable area.

scroll-half-page-down

Scrolls down by about half a page.

scroll-half-page-up

Scrolls up by about half a page.

hint-mode-current-tab

Shows link hints and opens the selected target in the current tab.

hint-mode-new-tab

Shows link hints and opens the selected link in a new tab.

hint-mode-right-click

Shows current-tab-style hints, but right-clicks the selected target instead.

reload-current-tab

Reloads the current page.

reload-current-tab-hard

Hard reloads the current page and bypasses cache.

find-mode

Opens the in-page find bar and previews the current query's match count.

watch-mode

Enters watch mode on the best visible video, or exits watch mode if already active.

yank-link-url

Shows link hints for visible links and copies the selected link URL to the clipboard.

yank-image

Shows hints for visible images and copies the selected image to the clipboard.

yank-image-url

Shows hints for visible images and copies the selected image URL to the clipboard.

yank-current-tab-url

Copies the current tab URL to the clipboard.

yank-current-tab-url-clean

Copies the current tab URL to the clipboard after removing common tracking query params like `si`, `utm_*`, and `fbclid`.

history-go-prev

Navigates back in the current tab's history.

history-go-next

Navigates forward in the current tab's history.

tab-go-prev

Activates the previous tab in the current window.

tab-go-next

Activates the next tab in the current window.

first-tab

Activates the first tab in the current window.

last-tab

Activates the last tab in the current window.

create-new-tab

Opens a new browser tab.

close-current-tab

Closes the active browser tab.

close-tabs-other

Closes every other tab in the current window except the active tab.

close-tabs-left

Closes tabs to the left of the active tab in the current window.

close-tabs-right

Closes tabs to the right of the active tab in the current window.

move-tab-left

Moves the active tab one position left within its current pinned state group.

move-tab-right

Moves the active tab one position right within its current pinned state group.

restore-closed-tab

Restores the most recently closed tab or window.

visit-previous-tab

Activates the most recently visited tab in the current window.

duplicate-current-tab

Duplicates the active browser tab.

duplicate-current-tab-origin

Opens a duplicate of the active browser tab using only its origin, without the path, query string, or hash.

move-current-tab-to-new-window

Moves the active browser tab into a new window.

bar-mode-current-tab

Opens the bar prompt and loads the selected URL or search in the current tab.

bar-mode-new-tab

Opens the bar prompt and loads the selected URL or search in a new tab.

bar-mode-edit-current-tab

Opens the bar prompt with the current tab URL prefilled for quick editing.

In find mode:

cycle-match-next

Moves to the next in-page search match while find mode is active.

cycle-match-prev

Moves to the previous in-page search match while find mode is active.

Toggles fullscreen for the active watch-mode video.

toggle-play-pause

Toggles play/pause for the active watch-mode video and exits watch mode.

toggle-loop

Toggles loop playback for the active watch-mode video.

toggle-mute

Toggles mute for the active watch-mode video.

toggle-captions

Toggles captions for the active watch-mode video when available.

hints

hints.showCapitalizedLetters

Changes how hint labels are displayed. The required key presses stay lowercase.

hints.improveThumbnailMarkers

Makes hint markers much easier to see on thumbnail-like media targets by centering them and rendering a larger badge.

hints.activationIndicator.enabled

Shows a temporary activation ring after you open or focus something through hints.

hints.activationIndicator.color

Sets the activation ring color used when `hints.activationIndicator.enabled` is on.

hints.charset

Controls which letters are used to generate hint labels. Letters earlier in the charset are prioritized first.

This applies to generated (non-reserved) hint labels. Reserved directive labels can still be typed even if they use letters outside this charset.

Use letters only, with no spaces and no duplicates. Any repeated character, whitespace, or non-letter is treated as an error.

hints.minLabelLength

Sets the minimum number of characters generated hint labels must use.

hints.avoidAdjacentPairs

Lists consecutive character pairs to avoid inside generated hint labels. It is recommended to put hard-to-press character pairs here.

Separate pairs with spaces or newlines. Entries must be exactly two letters, and `#` starts a comment anywhere on the line.

hints.directives

Sets preferred hint directives for special elements.

Rules:
- Each directive must be on its own line.
- Use exactly one space between the directive and labels, and between labels.
- Labels must contain only letters a-z and each next label must be exactly one character longer than the previous label.
- Matching directive hints show the directive icon next to the hint label.
- Directives can use letters outside `hints.charset` and are still typeable in hint mode.

hints.advanced.customSelectors

Adds site-specific selector overrides for hint generation.

This option can override the default hint target discovery on sites where important interactive surfaces are not exposed as normal clickable HTML controls.

It is especially useful for applications with custom chat inputs, editors, or button-like surfaces that rely on framework-managed click handling instead of standard browser semantics.

Matching selectors allow Nav to surface those targets in hint mode so they can still be addressed and activated with a click event.

Rules:
- Each site block must start with `* <regex-pattern> {` and end with `}`.
- `regex-pattern` matches `window.location.href`.
- Each entry inside the block must be `<key> <css-selector>`.
- `key` must be letters only and at least as long as `hints.minLabelLength`, or exactly `<auto>`.
- `<auto>` keeps the selector site-specific but lets the label be generated from `hints.charset`.
- Explicit keys reserve that label and override the normal generated label for the first visible matching element.
- Custom selector matches are included in hint generation even when the page would not normally expose them as hintable targets.

* ^https://chatgpt\.com/ {

  proj button[data-testid='create-new-project-button']

  <auto> nav[aria-label='Chat history'] a

}

hints.styling

Chooses how hint markers are styled.

`default` uses the built-in marker look.
`custom` uses the CSS in `hints.customCSS`.

hints.customCSS

Custom CSS applied to hint markers when `hints.styling` is set to `custom`.

bar.color

Controls the outline color of the bar prompt.

bar.searchEngineURL

Used when the bar input is not recognized as a URL.

Use `{}` to represent the search query.
Example: `https://www.google.com/search?q={}`
